1.Files

  • 文件是保存数据的地方

    • 构建文件对象

      • new File(String pathName) //根据路径构建一个File对象

      • new File(String parent, String Child) //根据父目录+子路径构建

    • 操作文件

      • boolean exists()：判断此路径是否真实存在
      • boolean isFile()：判断是否是文件
      • boolean isDirectory()：判断是否是目录
      • long length()：查看文件的大小，字节量
      • String getAbsolutePath()：获取绝对路径
      • String getName()：获取名称(文件名称，目录名称)
      • String[] Arrays.toString(file.list())：获取所有的子项，返回所有子项的名称
      • File[] listFiles()：获取所有子项，返回所有子项的File对象的数组

            public static void deleteFile(File file){

  File[] files = file.listFiles();

      if(f.isDirectory()){//是目录

            public static long countSize(File file){

              if(!file.exists()) return 0;
  long size = 0;
  File[] files = file.listFiles();
  if(files != null) {
      for (File f : files) {
          if (f.isDirectory()) {//递归统计
              size += countSize(f);
          } else {
              size += f.length();
          }
      }
  }
  return size;

            }
            //获取某个目录下所有的.exe程序，打印文件名
            public static void getExe(File file){

              if(!file.exists()) return;
  File[] files = file.listFiles();
  if(files != null){
      for(File f:files){
          if(f.isDirectory()){
              getExe(f);//递归进行
          }else{
              String name = f.getName();
              if (name.endsWith(".exe")){
                  System.out.println(f.getAbsolutePath());
              }
          }
      }
  }

            }
            }

2.4 字节流 FileInputStream 和 FileOutputStream

2.4.1 概念

  • 以字节为单位操作数据
  • 内部默认也提供的缓冲区
  • 所的字符流都是基于这两个字节流的拓展
  • 更适合操作字节文件

2.4.2 常用方法

  • FileInputStream

    • 序号 方法及描述
      1 public void close() throws IOException{} 关闭此文件输入流并释放与此流有关的所有系统资源。抛出IOException异常。
      2 protected void finalize()throws IOException {} 这个方法清除与该文件的连接。确保在不再引用文件输入流时调用其 close 方法。抛出IOException异常。
      3 public int read(int r)throws IOException{} 这个方法从 InputStream 对象读取指定字节的数据。返回为整数值。返回下一字节数据，如果已经到结尾则返回-1。
      4 public int read(byte[] r) throws IOException{} 这个方法从输入流读取r.length长度的字节。返回读取的字节数。如果是文件结尾则返回-1。
      5 public int available() throws IOException{} 返回下一次对此输入流调用的方法可以不受阻塞地从此输入流读取的字节数。返回一个整数值。

  • FileOutputStream

    • 序号 方法及描述
      1 public void close() throws IOException{} 关闭此文件输入流并释放与此流有关的所有系统资源。抛出IOException异常。
      2 protected void finalize()throws IOException {} 这个方法清除与该文件的连接。确保在不再引用文件输入流时调用其 close 方法。抛出IOException异常。
      3 public void write(int w)throws IOException{} 这个方法把指定的字节写到输出流中。
      4 public void write(byte[] w) 把指定数组中w.length长度的字节写到OutputStream中。

2.4.3 案例

  • FileOutputStream

    • ```java
      public class TestCase07 {
      public static void main(String[] args) throws Exception {
    FileOutputStream out = new FileOutputStream("E:\\b.txt");

2.5.4 处理流-BufferedReader 和 BufferedWriter

  • BufferedReader和 BufferedWriter属于字符流，是按照字符来读取数据的

  • 字符流，为其他的字符流提供缓冲更大的缓冲区

  • 默认的缓冲区大小是16KB，可以指定缓冲区的大小

  • 最大的好处是可以按行读取数据

  • 不能单独存在，缓冲流存在的意义就是为其他的流提供缓冲区

  • 以字符为单位操作数据

  • 关闭时处理流，只需要关闭外层流即可

  • 使用

2.7 对象流 ObjectInputStream&ObjectOutputStream

2.7.1 概念

  • 序列化和反序列化
    • 序列化： 将内存中的java对象转为字节数据的过程，持久保存内存中的对象，做存档操作
    • 反序列化：将字节数据转为对象的过程
  • 对象流总结：
    • 对象能够被序列化的前提是实现序列化接口
    • 如果对象的某个属性不想被序列化，可以使用transient关键字来修饰，所有的集合在序列时，会忽略掉集合中的数据
    • 序列化时不要忘记添加serialVersionUID，如果没有添加serialVersionUID，在对象发生改变时，会导致反序列化失败
    • 同时序列化多个对象时，应在文件的末尾多写入一个null,可以有效的避免EOFException的发生

2.7.2 案例

  • ```java
    public class TestCase06 {
    public static void main(String[] args) throws Exception {
    Student s1 = new Student(23,"tom");
    Student s2 = new Student(25,"jack");
    Student s3 = new Student(28,"rose");
    //构建对象流时需要指定一个文件字节流
    ObjectOutputStream out = new ObjectOutputStream(new FileOutputStream("E:\\student.obj"));
    out.writeObject(s1);//序列化操作
    out.writeObject(s2);
    out.writeObject(s3);
    out.writeObject(null);//这个操作是为了预防EOFException
    out.close();
    System.out.println("序列化成功！！");
}
    }
    //对象能够被序列化的前提是要实现 Serializable标志接口

2.9 I/O流 数据传输的机制

  • FileWriter/FileReader 以字符为单位操作数据，专门针对文本文件
  • FileInputStream/FileOutputStream 以字节为单位操作数据，是最基本的字节流，是所有流的底层
  • BufferedReader/BufferWriter 以字符为单位操作数，提供更大的缓冲区，可以按行读取文件,不能单独存在，需要基于其他的字符流来工作
  • OutputStreamWriter/InputStreamReader 转换流，核心意义就是提供字节流与字符流转换的功能，转换时可以指定编码格式，以字符为单位操作数据
  • ObjectInputStream/ObjectOutputStream 对象的序列化和反序列化操作
  • PrintStream/PrintWriter 打印流，提供了很多重载的打印方法，可以很方便的打印各种类型的数据
  • 选择IO的原则：字节文件尽量使用字节流来操作
