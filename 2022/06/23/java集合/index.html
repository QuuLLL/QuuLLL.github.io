1.Collection

1.1 概念

  • 代表集合的顶级父接口，定义了集合中常见的操作的方法
  • 它是所有集合的根，分别有List(有序集),Set(无序集),Queue(队列)三类实现
  • 集合可以看成是存储数据的容器
  • 集合可以存储不同数据类型的数据，但在实际应用中，我们通常只存储一种数据类型的数据

1.2 常用方法

  • boolean add()：添加元素
  • boolean addAll()：集合合并
  • boolean remove(): 删除元素
  • boolean removeAll(): 根据子集来删除
  • boolean contains(): 是否包含指定的元素
  • boolean containsAll(): 是否包含指定的子集合
  • int size(): 查看集合的大小
  • boolean isEmpty(): 是否是一个空集合
  • void clear(): 清空集合

2. List 有序集

2.1 概念

  • List是Collection的子接口，具备Collection定义的常用操作
  • List代表有序集，元素是有序(存取顺序)
  • 可以通过下标来随机访问元素，下标从0开始，到size-1位置结束, 使用下标访问元素时，必须是合理的下标，否则将抛出IndexOutOfBoundsException
  • 允许重复的元素存在
  • List是一种线性顺序的数据结构

2.2 使用案例

2.3 ArrayList

  • 底层维护这一个Object类型的数组，用于存储数据,数组的默认长度是10

  • 这个数组有自己的扩容机制，当容量不够时，自动扩容为原容量的1.5倍，最大能扩容到Integer.MAX_VALE

  • ArrayList是List接口后期的实现，非线程安全的实现，可以通过Collections辅助工具转为线程安全的集合

  • 可以调用有参构造为ArrayList指定初始容量

  • 因为是数组实现，查询效率高，而增删元素效率较低

 * @param e
 * @param index
 * @return
 */
public boolean add(E e,int index){
    checkIndex(index);
    checkCapacity(size+1);
    for(int i=size+1;i>=index;i--){
        data[i] = data[i-1];
    }
    data[index] = e;
    return true;
}
/**
 * //检查容量
 * @param minCapacity 最小需要容量
 */
private void checkCapacity(int minCapacity){
    if(minCapacity>data.length){//容量不够了,扩容
        int oldCapacity = data.length;
        int newCapacity = oldCapacity+(oldCapacity>>1);//扩容1.5倍
        if(newCapacity-minCapacity<0){
            newCapacity = minCapacity;
        }
        data = Arrays.copyOf(data,newCapacity);//扩容
    }
}

    checkIndex(index);

3. Set

3.1 概念

  • 元素无序(存储顺序),特别是不保证元素的恒定顺序
  • 元素不允许重复(equals为标准)
  • 没有下标，不能通过下标来访问元素

3.2 常用方法

  • boolean add(E e)：如果指定的元素尚不存在，则将其添加到此集合（可选操作）。
  • boolean addAll(Collection<? extends E> c)：如果指定集合中的所有元素尚未存在（可选操作），则将其添加到此集合中。
  • void clean()：从该集合中删除所有元素（可选操作）。
  • boolean contains(Object o)： 如果此set包含指定的元素，则返回 true 。
  • boolean containsAll(Collection<?> c)： 如果此集合包含指定集合的所有元素，则返回 true 。
  • boolean equals(Object o)：将指定对象与此set进行相等性比较。
  • int hashCode()： 返回此set的哈希码值。
  • boolean isEmpty()：如果此集合不包含任何元素，则返回 true 。
  • boolean rmove(Object o)：如果存在，则从该集合中移除指定的元素（可选操作）。
  • boolean removeAll(Collection<?> c)：从此集合中删除指定集合中包含的所有元素（可选操作）。
  • boolean retainAll(Collection<?> c)： 仅保留此集合中包含在指定集合中的元素（可选操作）。
  • in size()： 返回此集合中的元素数（基数）。
  • Object[] toArray()： 返回包含此set中所有元素的数组。

3.3 案例

  • ```java

3.5 TreeSet

  • TreeSet是Set的实现类，可以保证元素的自然排序

  • TreeSet底层是一个TreeMap

  • 默认情况下，Tree是利用元素的排序规则来进行排序，这就要求

    • 被添加到TreeSet中的元素实现比较接口Comparable接口，指定比较规则
    • 如果没有指定比较规则，会发生运行错误

  • 如果元素没有实现比较规则，可以在构建一个TreeSet时指定一个比较器Comparator

  • 案例：

    • ```java
      public class TestCase03 {

      public static void main(String[] args) {
    TreeSet<Integer> set = new TreeSet<>();
    set.add(18);
    set.add(15);
    set.add(23);
    set.add(45);
    set.add(38);
    set.add(18);
    System.out.println(set);

    TreeSet<Student> set02 = new TreeSet<>();
    //set02.add(new Student(1001,"AA"));
   // set02.add(new Student(1002,"BB"));
    System.out.println(set02);
    set02 = new TreeSet<>(new Comparator<Student>() {
        @Override
        public int compare(Student o1, Student o2) {
            return o1.getNo()-o2.getNo();
        }
    });
    set02.add(new Student(1003,"CC"));
    set02.add(new Student(1001,"AA"));
    set02.add(new Student(1002,"BB"));
    System.out.println(set02);

    //特有的方法
    Integer first = set.first();//获取排名第一的那个元素
    System.out.println(first);//15
    Integer last = set.last();//获取排名最后的那个元素
    System.out.println(last);
    Integer integer = set.pollFirst();//获取第一个元素并删除
    Integer integer1 = set.pollLast();//获取最后的元素并删除
    Integer floor = set.floor(40);//获取比指定数据更小的那个元素
    System.out.println(floor);

    //遍历TreeSet
    //迭代遍历
    Iterator<Integer> iterator = set.iterator();
    //循环删除的方式来遍历
    while (set.size()>0){
        first = set.pollFirst();
        System.out.println(first);
    }
}

4 . TreeMap

  • 排序的Map,按key自动排序(升序 )
  • 存储在TreeMap中的元素必须实现比较接口
  • 也支持在构建TreeMap时，指定比较器

5. LinkedHashMap

  • 保证元素的存储顺序一致，还具备HashMap的所有特性

队列

  • 队列是一种先进先出的数据结构

    • 队头：最先被添加的元素
    • 队尾：最后添加的元素

  • 案例

    • public class TestCase06 {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();
