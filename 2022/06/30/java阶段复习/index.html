数据类型

  • 基本数据类型

    类型 范围 字节数 说明
    byte -2^7~2^7-1 1字节 字节最小单位
    short -2^15~2^15-1 2字节
    int -2^31~2^31-1 4字节 整数字面量默认int类型
    long -2^63~2^63-1 8字节 长整形
    float 4字节 小数字面量要表示float需要加F|f后缀
    double 8字节 小数字面量默认是double类型
    char 0~65535 2字节 字符实际上也是一个整数
    boolean true/false 1字节

    问：float是4字节32位，int也是4字节32位，为什么float比int大？

    答：因为整数与浮点数的存储形式是不一样的，浮点数采用科学计数法来存储数据

    浮点数会把32位二进制拆分成 4个部分 符号位，指数位，幂，小数位

基本数据类型所遵循的规则：

1.小类型转为大类型，自动类型转换

2.大类型转为小类型，强制类型转换 ，有可能会有溢出问题

3.在运算时，byte,short,char会自动转为int计算

java面向对象

在程序设计中，将一切 东西都看成是对象

类

某一类抽象事物的概念，是一个名词，用class描述 ，成员变量描述事物的属性，方法描述事物的行为

对象

由类实例化出来的真实存在的个体 ，对象初始化后，它的属性都具有默认值:

int–>0,double–>0.0,boolean–>flaise,引用类型–>null

面向对象三大特征

  • 封装：隐藏对象的内部属性和实现细节 ，封装的原则：属性私有，提供公开的访问属性setter/getter方法

  • 继承：当子类继承父类后，子类能够吸收父类的功能，并具备拓展父类的功能的能力，子类拓展父类的功能：

    a.重写父类的方法，b.添加子类特有的方法

    • 继承不能选择性的继承
    • private final修饰的方法或属性，子类不能继承到
    • private final,static 修饰的方法，没有方法重写
    • 继承具有单根性，一个子类只能有一个直接父类，可以有多个间接父类
    • 继承也具有向下传递的效果

  • 多态：同一个对象在不同时刻，可以具备多个形态

    • 核心机制：父类的引用可以指向子类的对象
    • 向上造型：多态的一种形式，Person p = new Student(),同一调用规范
    • 向下造型：将父类的引用造型为子类的类型，使用子类特有的功能，向下造型时应使用instanceof关键字判断是否能造型
    • 多态的调用规制：能调用什么，看引用的类型，调用方法，看实际指向的对象类型，静态方法与引用绑定，与对象无关

构造方法

构造方法是一种特殊的方法，用于创建对象的方法，与类同名，没有返回值，也不用void修饰

  • 如果一个类没有提供构造方法，编译器会自动添加一个无参构造
  • 如果明示提供了构造方法，编译器不再提供默认的无参构造，此时调用无参构造，会编译失败
  • 构造方法的第一行默认是super()，调用父类的构造方法
  • this()代表调用本类的其他构造方法，this()与super()互斥
  • 构造方法的作用：a.创建对象，b.对对象的属性做初始化操作

this与super

  • this指代当前对象，谁来调用，this指代谁
  • super指代父类对象的引用，如果一定要使用父类的属性或者方法， 可以使用super.来明确调用
  • 当成员变量与局部变量同名时，可以使用this.来区分

static&final

static 代表静态，可以修饰类，方法，属性，只有内部类可以使用static修饰

static属于类，会优先于对象加载，且只加载一次，全局唯一，全局共享，一些共享资源可以使用static

final 最终的

修饰类：类不能再进行拓展，不能被继承

修饰方法：方法不能再重写

修饰变量：变量不能被二次赋值

问：static方法与实例方法的区别?

答：静态方法中不能直接调用实例方法，实例方法中可以直接调用静态方法

​ 因为静态方法属于类，优先于对象加载，静态方法中没有隐式的this，

抽象类与接口

  • 抽象类：包含抽象方法类，称为抽象类

  • 接口：接口中只有抽象方法，接口中变量默认就是常量

问：接口与抽象类区别?

答：1.抽象类更多用于抽象多个类的共有属性和方法，定义一个共同的父类，接口更多用于定义调用的规范

​ 2.抽象类中可以包含普通方法，接口中都是抽象方法

​ 3.抽象类可以定义构造方法，接口中不能定义构造方法

​ 4.抽象类可以有普通的成员变量，接口中只有常量

​ 5.抽象类只能单继承，接口可以多继承

问：方法重写与方法重载的区别？

答：重载：在一个类中，某几个方法的方法名称相同，参数列表不同，方法重载的意义在于统一调用规则

​ 重写：存在于继承关系中，子类的某个方法与父类的方法方法定义完全一致，相当于子类覆盖掉了父类的这个方法，方法重写的意义在于覆盖掉父类的方法，进行拓展

方法重写所遵循的规则：

  1. 子类方法名称必须与父类相同
  2. 子类方法参数列表必须与父类相同
  3. 子类方法的访问权限必须大于或等于父类
  4. 子类方法抛出的异常必须小于或等于父类
  5. 子类方法的返回值类型必须与父类一致

String类

String是final修饰的类，代表不可变字符串，调对字符的操作都会生成新的字符串对象，字符串的内部维护着一个常量池，使用字面量值创建的字符串对象都存储在常量池中，这样做的意义是为了字符串对象的复用

问：String与StringBuffer/StringBulider的区别？

答：1.String是不可变字符，对其操作都会生成新的字符串对象，StringBuffer是可变字符串，对其操作不会生成新的字符串对象，都是操作的自身，每个方法返回都是this

2.频繁修改字符串时，应选择StringBuffer ,如果使用String，效率会降低

3.StringBuffer是早期实现，线程安全的实现，StringBuilder是后期的实现，是非线程安全的实现

4.String有重写equals方法，StringBuffer/StringBuilder没有重写equals方法

包装类

  • 自动装箱 将基本数据类型的字面量直接赋值给包装类型

  • 自动拆箱 将包装类型的引用直接赋值给基本类型的数据

日期类

  • Date 封装一个时间毫秒值
  • SimpleDateFormat 日期与字符串转换工具类
  • Calendar 代表日历，比Date封装了更多的信息

集合框架

Collection 集合框架的顶级父接口，有List,Set,Queue三个子接口

List

  • 有序集，元素有序(存取顺序)
  • 可以包含重复元素
  • 线性顺序结构，可以通过下标随机访问元素

Set

  • 无序集，元素无序(存取顺序)
  • 不能包含重复的元素
  • 不能通过下标来访问元素

Queue

  • 代表队列，类似与管道，元素先进先出
  • FIFO
  • 队首，队尾

ArrayList

  • 数组实现的List
  • 数组的初始长度是10
  • 数组扩容机制：当容量不够时，自动扩容为原容量的1.5倍，最大能扩容到Integer.MAX_VALUE-8
  • ArrayList是一个非线程安全的实现，但可以通过Collections工具类的方法转为线程安全的集合
  • 因为是数组实现，查询效率高，而增删元素效率相对较低

LinkedList

  • 链表实现的List，是一个双向链表
  • 在存储数据时，会先将数据转为一个节点(Node)
  • LinkedList虽然是链表实现，但其内部也额外维护着一个下标，可以同下标来操作
  • LinkedList是链表实现，查询效率低，但增删元素效率高，操作头尾节点效率高
  • LinkedList也是Queue接口的实现类，也可以将LinkedList当成队列来处理

Vector

  • 数组实现的List
  • 数组的初始长度是10
  • 数组的扩容机制：如果有指定增量，扩容：原容量+增量，如果没有指定增量,扩容为原容量的2倍
  • Vector是早期实现，是一个线程安全的实现，效率相对较低

Stack

  • 是Vector类的子类
  • 是一个栈结构，元素先进后出 FILO
  • 压栈：将元素压入栈顶
  • 弹栈：获取栈顶元素并删除

HashSet

  • 内部维护着一个HashMap
  • 实质上就是一个没有value值的HashMap

TreeSet

  • 内部维护着一个TreeMap
  • 实质上就是一个没有value值的TreeMap
  • 可以保证元素的自然排序
  • 因为要保证自然排序，添加TreeSet中的元素必须指定排序规则(实现Comparable接口)
  • 如果元素没有实现比较接口，也可以在构建TreeSet时，指定比较器(Comparator)

LinkedHashSet

  • 是HashSet的子类
  • 能保证元素的存取顺序，同时具备HashSet的特性

Map

代表映射

以键值对的形式存储数据

可以通过键（key)获取值(value)

key不允许重复，value可以重复

null值可以作为key,也可以作为value

如果添加相同的key，会替换原有的数据

HashMap

  • 数组+链表的实现
  • 数组的长度默认是16
  • 扩容机制：当容量到达原容量的3/4时，自动扩容，扩容为原容量的2倍
  • 存储元素是，先计算元素在数组中存放的位置，如果这个位置上没有元素，直接存储，如果有元素，依次equals比较，有true的情况，替换掉原有的值，没有true的情况，以链表的形式存储数据
  • 如果数组的长度超过64，且链表的长度超过8时，链表会自动转为红黑树
  • 在构建HashMap时，可以指定数组的初始容量，初始容量为离指定容量最近的一个2的n次幂，例如：指定容量为5,那个HashMap的初始容量为8，这样做的目的是为了保证其内部的计算存储元素位置的算法恒成立
  • 如果要保证HashMap的效率，内容相同的对象，hashCode一定要相同，且equals相同
  • 优化HashMap的建议:a.结合要存储的数据的总量，合理计算HashMap的初始容量及加载因子，b.合理重写hashCode和equals方法
  • HashMap是非线程安全的实现
  • HashMap不保证元素的存取顺序，特别是不保证恒定顺序，当数组扩容时，HashMap的所有元素会rehash(重排列)

HashTable

  • 与HashMap结构相同，但不会转为红黑树
  • HashTable是早期实现，是线程安全的实现

TreeMap&LinkedHashMap

  • TreeMap 保证顺序(排序),key值必须实现比较接口
  • 构建TreeMap时，指定比较器
  • LikedHashMap是HashMap的子类，可以保存元素的存储顺序

Map的遍历方式

  • Set keySet()
  • Set entrySet()
  • Collection values()

I/O流

I/O流实质上就是一种数据传输的机制

流的分类

按传输的方向

输入流 in

输出流 out

按传输的方式

字符流 writer/reader 以字符为单位读写数据

字节流 outputStream/InputStream 以字节为单位读写数据

  • FileWriter/FileReader 字符流 针对文本文件，默认缓冲区大小8kb
  • FileInputStream/FileOutputStream 字节流 以字节为单位读写数据，是所有流的根本
  • BufferRead/BufferWriter 缓冲流，为其他流提供更大的缓冲区，可以按行读写
  • InputStreamReader/OutputStreamWriter 转换流，提供字节转字符的编码支持
  • PrintWriter/PrintStream 打印流， 提供很多重载的打印方法，可以很方便的打印各种数据类型的数据
  • ObjectInputStream/ObjectOutputStream 序列化/反序列化

序列化/反序列化

  • 对象如果要支持序列化，必须实现序列化接口
  • 如果某个属性不想被序列化，需要使用transient修饰
  • 序列后，对象如果发生变化，反序列化会失败，为对象添加序列化的版本号可以解决这个问题
  • 静态的属性不会被序列化
  • 集合对象本身可以被序列化，集合中的数据不被会序列化
