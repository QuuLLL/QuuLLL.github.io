1.2.3 Object clone(Object obj)

  1. 克隆对象的副本，生成一个新对象

  2. 对象能被克隆的前提是，必须实现Cloneable接口

  3. 深克隆与前克隆

    1. 浅克隆
      • 克隆出来的两个对象可能不是两个完全独立的对象，可能持有与相同的引用
    2. 深克隆
      • 克隆出来的对象与原来的副本是连个完全独立的对象

3. StringBuffer && StringBuilder

  • StringBuffer && StringBuilder 为可变字符串

  • StringBuffer 是可变字符串，对其操作不会生成新的字符串，改变的都是他本身

  • StringBuffer 内部维护了一个默认长度为16的字符数组，当容量不够时会自动扩容，最大容量为Integer.MAX_VALUE-8

  • StringBuffer 没有重写equals方法

  • StringBuffer是早期的实现，是线程安全的实现，StringBuilder是后期的实现，非线程安全的实现

  • 当涉及字符串频繁修改时，应选择StringBuffer提升效率

  • StringBuffer类是对String类的补充

  • 常用方法

    • public StringBuffer append(String s)：将指定的字符串追加到此字符序列。
    • public StringBuffer reverse()：将此字符序列用其反转形式取代。
    • public delete(int start, int end)：移除此序列的子字符串中的字符。
    • public insert(int offset, int i)：将 int 参数的字符串表示形式插入此序列中。
    • insert(int offset, String str)：将 str 参数的字符串插入此序列中。
    • replace(int start, int end, String str)：使用给定 String 中的字符替换此序列的子字符串中的字符。（[)）

5.4 Random类

5.4.1 概念

  • Random类是java.until下的一个根据随机算法的起源数字进行一些变化，从而得到随机数字的方法。
  • 随机算法的起源数字被成为种子数（seed）。
  • 虽然Random类产生的数字是随机的，但在相同种子数（seed）下的相同次数产生的随机数是相同的（伪随机）。

5.4.2 常用方法

  • random.nextInt(): 返回值为整数,范围是int类型范围
  • random.nextLong(): 返回值为长整型，范围是long类型的范围
  • random.nextFloat(): 返回值为小数，范围是[0,0.1]
  • random.nextDouble(): 返回值为小数，范围是[0,0.1]
  • random.nextBoolean(): 返回值为boolean值，true和false概率相同

5.5 Calender

5.5.1 概念

  • 日历类 Calendar
  • 封装了日历数据，比Date封装信息更丰富
  • 日历类的默认实现 格里高利历

5.5.2 Calendar类对象字段类型

  • Calendar.YEAR：年份
  • Calendar.MONTH：月份
  • Calendar.DATE：日期
  • Calendar.DAY_OF_MONTH：每月的第几天
  • Calendar.HOUR：12小时
  • Calendar.HOUR_OF_DAY：24小时
  • Calendar.DAY_OF_WEEK：星期几
